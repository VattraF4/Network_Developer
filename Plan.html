Core Networking

This section covers the foundational Layer 2 and Layer 3 configurations. You'll set up Virtual LANs (VLANs) to segment the network, configure a router to enable communication between them (Inter-VLAN Routing), and implement DHCP to automate IP address assignment for end devices.

Day 1-2: VLANs & Inter-VLAN Routing

Goal: Set up 4 VLANs with Router-on-a-Stick.

1. Create VLANs on a Layer 2 Switch:

Switch(config)# vlan 10
Switch(config-vlan)# name StudyOffice
Switch(config)# vlan 20
Switch(config-vlan)# name StudentSupport
Switch(config)# vlan 30
Switch(config-vlan)# name HumanResource
Switch(config)# vlan 40
Switch(config-vlan)# name ITCenter

2. Assign Access Ports:

Switch(config)# interface fa0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10

3. Configure Trunk to Router:

Switch(config)# interface fa0/24
Switch(config-if)# switchport mode trunk

4. Router-on-a-Stick:

Router(config)# interface g0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0
Day 3-4: DHCP Configuration

Goal: Configure DHCP for all VLANs.

1. Set up DHCP on Router:

Router(config)# ip dhcp excluded-address 192.168.10.1
Router(config)# ip dhcp pool VLAN10_Pool
Router(dhcp-config)# network 192.168.10.0 255.255.255.0
Router(dhcp-config)# default-router 192.168.10.1

2. Verify PCs get IPs: Use `ipconfig` on client PCs.

Routing

This section focuses on how data is forwarded between different networks. You will first implement simple static routes for predictable paths and then transition to a dynamic routing protocol, OSPF, which allows the network to automatically adapt to changes and scale more effectively.

Day 3-4: Static Routing

Goal: Configure static routing between the main campus and a remote site.

1. Configure Static Route:

Router1(config)# ip route 192.168.20.0 255.255.255.0 10.0.0.2
Day 5-6: OSPF Routing

Goal: Replace static routes with OSPF for scalability.

1. Enable OSPF on Routers:

Router(config)# router ospf 1
Router(config-router)# network 192.168.10.0 0.0.0.255 area 0

2. Verify OSPF Neighbors with `show ip ospf neighbor`.

Wireless & Redundancy

Here, you'll expand the network to include secure wireless access and build in fault tolerance. This involves configuring a Wireless LAN Controller (WLC) with enterprise-grade security and implementing a First Hop Redundancy Protocol (HSRP) to ensure network uptime even if a primary router fails.

Day 7-8: Wireless LAN (WPA2 + MAC Filtering)

Goal: Secure Wi-Fi for students/staff.

1. Configure a WLC 3504 in Packet Tracer.

2. Set up a WPA2-Enterprise SSID, which requires a RADIUS server.

3. Enable MAC Address Filtering to whitelist allowed devices.

Day 9-10: HSRP (First Hop Redundancy)

Goal: Ensure gateway redundancy.

1. Configure HSRP on R1 (Primary) & R2 (Backup):

R1(config-if)# standby 1 ip 192.168.10.254
R1(config-if)# standby 1 priority 150
R1(config-if)# standby 1 preempt

2. Test failover by shutting down the primary router's interface.

LAN Security

Securing the wired network is critical. This section focuses on implementing multiple layers of security. You will configure Port Security to control which devices can connect to switch ports, 802.1X for robust user authentication, and Access Control Lists (ACLs) to filter traffic and enforce communication policies between network segments.

Day 11-12: Port Security, ACLs, 802.1X

Goal: Prevent unauthorized access.

1. Configure Port Security on access ports:

Switch(config-if)# switchport port-security
Switch(config-if)# switchport port-security maximum 2
Switch(config-if)# switchport port-security violation restrict

2. Configure 802.1X for wired security using a RADIUS server.

Switch(config)# aaa new-model
Switch(config)# dot1x system-auth-control

3. Set up an ACL to block unauthorized subnets:

Router(config)# access-list 100 deny ip 192.168.10.0 0.0.0.255 192.168.30.0 0.0.0.255
Router(config)# access-list 100 permit ip any any
Router(config)# interface g0/0.10
Router(config-subif)# ip access-group 100 in

Finalization

The final phase involves comprehensive testing of the entire network to ensure all components work together seamlessly. After verification, you will compile your findings, configurations, and test results into a final report as outlined below. This is a critical step to demonstrate your work and document the project's success.

Day 13-14: Testing & Documentation

Final Report Structure:

  • Introduction: Project Scope, Objectives, Team Members, and Tools Used.
  • Configuration Steps: Detailed explanations with screenshots and command outputs for each major task.
  • Challenges Faced & Solutions: A log of any issues encountered and how they were resolved.
  • Verification: Evidence of functionality, including successful ping tests and `show` command outputs.
  • Conclusion: A summary of the project outcome and lessons learned.
